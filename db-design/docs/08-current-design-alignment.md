## 整體結論

原本 8 份資料庫規劃可以繼續作為資料層基礎，不需要推翻。

目前畫面設計比原規劃更具體，尤其新增了這幾個實際作業流程：

1. 預檢系統管理改為月曆入口。
2. 每月 Excel 預排表需要匯入並產生月曆排程。
3. 點選月曆排程後，需要自動帶入資料並準備列印。
4. P 單不是故障才有，預防檢修排定後就應建立 P 工單。
5. 作業前列印一次，完工後只做資料回填與履歷追蹤。
6. Word 檢查表格式不可改，正式列印應套用原始 Word 範本。
7. 物料畫面已分成物料概要與補料決策，避免兩者混在一起。
8. 清單維護需要成為所有下拉選單與基礎資料的後台。

因此現在需要做的是「補上預排、套版列印、回填結果」這幾塊資料設計，而不是重做原本的工單、物料、周轉件架構。

## 清單定義

目前系統內的「清單」應分成兩類，不應混在一起。

### 基礎清單

基礎清單是可被其他功能引用的主檔或選項資料，通常由清單維護後台管理。

範例：

| 清單 | 用途 | 建議資料來源 |
| --- | --- | --- |
| 車輛清單 | 預檢、故檢、履歷查詢 | `vehicle` 或 `equipment_group` |
| 機廠設備清單 | 機廠設備預檢 | `equipment_group` / `asset` |
| 人員清單 | 派工人、現場負責人、核准人員 | `person` |
| 維修單位清單 | 工單部門、成本中心 | `department` / `reference_option` |
| 檢修級別清單 | 第一級、第二級、第三級、第四級 | `pm_template` 或 `reference_option` |
| 安全項目清單 | 作業前列印勾選項 | `reference_option` |
| Word 範本清單 | 決定列印套用哪份原始檔 | 新增 `form_template` |
| 物料分類清單 | 物料查詢與補料分析 | `material` 欄位或 `reference_option` |

### 作業清單

作業清單是由實際流程產生的清單，不只是選項。

範例：

| 清單 | 定義 | 用途 |
| --- | --- | --- |
| 預排清單 | 每月 Excel 匯入後的預檢計畫列 | 產生月曆、審排、改期 |
| 月曆清單 | 預排清單依日期呈現 | 讓現場知道今天要做什麼 |
| 待列印清單 | 已建立 P 工單草稿但尚未列印 | 作業前核對與列印 |
| 回填清單 | 已列印或已執行，等待補實際資料 | 完工追蹤 |
| 履歷清單 | 已完成或異常追蹤的工單紀錄 | 查詢車輛、設備、物料歷史 |

## 工單定義

工單是一次實際要執行的工作，不是單純的清單資料。

目前工單總表仍應沿用舊規劃：

| 類型 | 名稱 | 定義 |
| --- | --- | --- |
| P | 預檢工單 | 預防檢修作業，包含車輛與機廠設備預檢 |
| C | 故檢工單 | 故障排除、現場拆裝、來源坑位 |
| R | 維修工單 | 拆下壞件後續內修、外修、驗收、回庫或報廢 |
| J | 專案工單 | 改善案、改造案、非例行批次工作 |

目前最需要先釐清的是 P 工單。

P 工單流程應定義為：

```text
預排項目 -> P 工單草稿 -> 作業前列印 -> 現場紙本檢查 -> 完工回填 -> 履歷追蹤
```

重要原則：

1. 預排項目還不是完整工單，它是每月排程來源。
2. 預排項目被點選或匯入確認後，可以建立 P 工單草稿。
3. P 單不是故障才有，預防檢修也要有 P 單。
4. 列印發生在作業前。
5. 回填發生在作業後。
6. 回填資料用於履歷與追蹤，不要求重新列印。

## 工單編號規則

工單號必須使用同一套正式規則，不可由畫面自行用車號、檢修級別或範本代碼拼出臨時格式。

正式格式：

```text
{TYPE}-{YYYMMDD}-{SITE}-{TARGET}-{SEQ}
```

範例：

```text
P-1150519-D-TS-001
P-1150519-D-TS-002
P-1150519-D-DE-001
```

欄位定義：

| 欄位 | 說明 |
| --- | --- |
| `TYPE` | 工單類型，例如 `P` 預檢、`C` 故檢、`R` 維修、`J` 專案 |
| `YYYMMDD` | 民國年月日，例如 `1150519` |
| `SITE` | 廠區，例如 `D` 淡海 |
| `TARGET` | 對象類別，例如 `TS` 列車、`DE` 機廠設備 |
| `SEQ` | 三碼流水號 |

流水號排序範圍：

```text
TYPE + YYYMMDD + SITE + TARGET
```

同一天的列車 P 工單與機廠設備 P 工單分開流水；P、C、R、J 也各自分開流水。

錯誤範例：

```text
P-11505-V-109-1Y
P-1150519-D-TS-109
P-11505-109
```

上述錯誤都把車號或級別放進工單流水位置，不能作為正式 P 單號。

### 2. Word 範本與欄位套版

建議新增：

| 資料表 | 用途 |
| --- | --- |
| `form_template` | 保存 Word 範本主檔資料與版本 |
| `form_template_field_mapping` | 定義 Word 欄位對應哪個系統欄位 |
| `work_order_print_job` | 保存每次產生 DOCX/PDF 或列印的紀錄 |

範本原則：

1. 原始 Word 檢查表不可改版面。
2. 系統只維護範本版本與欄位對應。
3. 正式列印應由後端套版輸出 DOCX 或 PDF。
4. HTML 預覽只能用來核對資料，不應取代正式 Word 格式。

| `pm_work_order_check_result` | 某張 P 工單的實際檢查結果 |

欄位可包含：

| 欄位 | 說明 |
| --- | --- |

| `result_value` | 實際數值或文字 |
| `result_status` | 正常、異常、未檢、N/A |
| `remark` | 備註 |

| `filled_at` | 回填時間 |

## 目前決策

目前正式方向如下：

1. 舊資料庫規劃保留，作為正式資料底層。
2. 目前前台設計以「月曆入口」作為預檢系統管理首頁。
3. 預排表匯入後先形成預排項目，再建立或關聯 P 工單。
4. P 工單可由預排自動建立草稿，不是故障才有。
5. 列印只發生在作業前。
6. 回填只做履歷、數值、用料、處理情形追蹤。
7. Word 檢查表格式不可重畫，必須用原始 Word 範本套版。
8. 清單維護是所有基礎資料與選項的後台，不是作業履歷本身。
