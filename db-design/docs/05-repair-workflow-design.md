# R 維修流程設計

## 定位

R 工單只處理「從車上或設備上拆下來的壞件」。

C 工單處理故障排除與現場拆裝，R 工單處理拆下壞件後續的維修、外修、驗收、回庫或報廢。

## 主狀態

R 工單主狀態放在 `work_order.status`，只保留 4 個：

| 主狀態 | 說明 |
| --- | --- |
| 待處理 | R 工單已建立，但尚未判定內修、外修或報廢。 |
| 維修中 | 已進入維修處理，包含內修、外修、驗收與報廢判定中。 |
| 已入庫結案 | 維修或驗收合格，設備已回到備品倉或可用庫存。 |
| 已報廢結案 | 已完成報廢判定與結案。 |

不要把已送修、已修回、驗收中、合格、退回全部塞進主狀態。這些放在 R 明細欄位。

## 明細欄位

| 清單 | 欄位 | 選項 |
| --- | --- | --- |
| 處理路線 | `repair_method` | 待判定、內修、外修、報廢 |
| 目前位置 | `current_place` | 待修區、內修區、廠商／採購驗收中、備品倉、報廢區 |
| 外修狀態 | `outsourcing_status` | 未送修、已送修、已修回、採購驗收中、採購驗收完成 |
| 驗收結果 | `acceptance_result` | 未處理、合格、退回、不合格 |
| 下一步 | `next_action` | 判定內修、外修或報廢、登錄維修結果、等待驗收完成、入庫結案、報廢結案、補拆下件序號或履歷、已結案 |
| 風險標籤 | `risk_tags` | 逾期、外修中、設備序號待補、履歷不完整、重複故障、高風險、建議報廢 |

選項由 `workflow_option` 管理。

## 標準流程

| 流程 | 主狀態 | 處理路線 | 目前位置 | 外修狀態 | 驗收結果 | 下一步 |
| --- | --- | --- | --- | --- | --- | --- |
| 建立 R 工單 | 待處理 | 待判定 | 待修區 | 未送修 | 未處理 | 判定內修、外修或報廢 |
| 判定內修 | 維修中 | 內修 | 內修區 | 未送修 | 未處理 | 登錄維修結果 |
| 判定外修 | 維修中 | 外修 | 廠商／採購驗收中 | 已送修 | 未處理 | 等待驗收完成 |
| 外修修回 | 維修中 | 外修 | 廠商／採購驗收中 | 已修回 | 未處理 | 等待驗收完成 |
| 驗收合格 | 維修中 | 外修或內修 | 廠商／採購驗收中 | 採購驗收完成 | 合格 | 入庫結案 |
| 回備品倉 | 已入庫結案 | 外修或內修 | 備品倉 | 採購驗收完成 | 合格 | 已結案 |
| 驗收退回 | 維修中 | 外修 | 廠商／採購驗收中 | 已送修 | 退回 | 等待驗收完成 |
| 判定報廢 | 已報廢結案 | 報廢 | 報廢區 | 未送修 | 不合格 | 報廢結案 |

## 看板分組

| 分組 | 推導方式 |
| --- | --- |
| 需判定 | 主狀態為待處理，或處理路線為待判定。 |
| 維修追蹤 | 主狀態為維修中，且不是等待驗收。 |
| 等待驗收 | 外修狀態為已修回或採購驗收中。 |
| 可入庫結案 | 驗收結果為合格，且尚未已入庫結案。 |
| 需補履歷 | 缺拆下件序號、來源坑位、拆裝事件或照片。 |
| 報廢處理 | 處理路線為報廢，或主狀態為已報廢結案。 |
| 結案查詢 | 已入庫結案或已報廢結案。 |

## 與設備履歷

每次 R 工單狀態變更，都應考慮是否新增 `asset_event`。

| R 動作 | asset_event |
| --- | --- |
| 建立 R 工單 | 下線待判定 |
| 判定內修 | 內修 |
| 送外修 | 送修 |
| 外修修回 | 修回 |
| 驗收合格回庫 | 回庫 |
| 報廢 | 報廢 |

## 外修紀錄

`repair_record` 保存外修細節：

| 欄位 | 說明 |
| --- | --- |
| `vendor_id` | 外修廠商。 |
| `rma_no` | 廠商維修編號。 |
| `send_date` | 送修日。 |
| `expected_return_date` | 預計回件日。 |
| `actual_return_date` | 實際回件日。 |
| `repair_result` | 維修結果。 |
| `cost_amount` | 維修費用。 |

## 報廢紀錄

`scrap_record` 保存報廢流程，R 工單只保存狀態與關聯，不應把所有報廢欄位塞進 `repair_work_order`。
