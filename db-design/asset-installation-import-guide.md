# 設備序號與坑位裝用匯入格式

這份格式給現場同仁盤點用。它先匯入暫存表，不會直接改正式設備裝用資料。

## 現場填寫檔

請使用：

```text
db-design/asset-installation-field-checklist-tamhai.csv
```

這份檔案已依淡海 18 台車、可裝設備坑位展開，共 3330 筆。欄位已改成中文，現場同仁主要填「設備序號」與「裝用狀態」。

## 欄位格式

| 欄位 | 必填 | 說明 |
| --- | --- | --- |
| `列號` | 是 | 流水列號，不要改 |
| `場站代碼` | 是 | `D` 淡海 |
| `對象代碼` | 是 | `TS` 列車 |
| `車號` | 是 | 車號，例如 `101` |
| `車廂模組` | 是 | 車廂/模組，例如 `M1` |
| `坑位代碼` | 是 | 坑位代碼，對應 `vehicle_position.position_code` |
| `坑位名稱` | 是 | 坑位名稱 |
| `坑位類型` | 是 | `獨立件`、`總成`、`子件` |
| `上層坑位代碼` | 否 | 上層坑位代碼 |
| `設備序號` | 視情況 | 設備銘牌序號；若 `裝用狀態=裝用中`，建議必填 |
| `現場設備名稱` | 否 | 現場看到的設備名稱，可用於比對 |
| `裝用狀態` | 是 | 裝用狀態 |
| `盤點人` | 建議 | 盤點人員 |
| `盤點日期` | 建議 | 盤點日期，格式 `YYYY-MM-DD` |
| `照片檔名或連結` | 否 | 照片檔名、雲端連結或附件編號 |
| `備註` | 否 | 備註 |

料號不用在這張表填，料號應由物料表與坑位/設備群組後續對應。

## 裝用狀態可填值

| 值 | 使用時機 |
| --- | --- |
| `裝用中` | 現場確認坑位上有設備 |
| `空坑` | 坑位存在，但目前沒有設備 |
| `查無銘牌` | 有設備，但找不到或看不清序號 |
| `與清單不符` | 現場設備與坑位名稱或預期不一致 |
| `待確認` | 尚未完成盤點 |
| `不適用` | 該坑位不適用目前車輛狀態 |

## 匯入流程

1. 現場同仁填寫 `asset-installation-field-checklist-tamhai.csv`。
2. 回收檔案後先匯入暫存表。
3. 系統檢查 `坑位代碼` 是否存在、`裝用中` 是否有填 `設備序號`。
4. 你確認後，再把通過的資料轉成正式 `asset` 與 `vehicle_position.current_asset_id`。

## psql 匯入指令

在 `db-design` 資料夾執行：

```powershell
psql "postgresql://root:YOUR_PASSWORD@YOUR_HOST:YOUR_PORT/zeabur" -v import_batch=TAMHAI-FIELD-CHECK-001 -f .\asset-installation-import-stage.sql
```

匯入後請先看檢查結果，不要直接套用正式資料。

## 照片處理

CSV 不能直接放照片檔本體，目前先填「照片檔名或連結」。

建議照片命名：

```text
坑位代碼_設備序號_盤點日期.jpg
```

例如：

```text
50-D-TS101-M1-BCU-AIO_BCU-AIO-001_2026-05-17.jpg
```

正式系統後續會再做附件上傳功能，讓每個坑位或設備序號可以掛多張照片。
