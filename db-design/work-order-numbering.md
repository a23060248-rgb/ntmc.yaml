# 工單編碼規則

正式格式固定為：

```text
{TYPE}-{YYYMMDD}-{SITE}-{TARGET}-{SEQ}
```

範例：

```text
C-1150514-D-TS-031
```

## 欄位定義

| 區段 | 範例 | 定義 |
| --- | --- | --- |
| TYPE | C | 工單性質 |
| YYYMMDD | 1150514 | 民國日期，115 年 05 月 14 日 |
| SITE | D | 場站，D=淡海，K=安坑 |
| TARGET | TS | 對象類別，TS=列車 |
| SEQ | 031 | 當日流水號，三碼補零 |

## 工單性質

| 代碼 | 名稱 | 用途 |
| --- | --- | --- |
| P | 預檢工單 | 週期性、計畫性保養檢查 |
| C | 故檢工單 | 現場故障排除、故障確認、拆換來源 |
| R | 維修工單 | 故障設備下線後的內修、外修、驗收、入庫或報廢 |
| J | 專案工單 | 改善案、改造案、批次專案、非例行工作 |

## 場站代碼

| 代碼 | 名稱 |
| --- | --- |
| D | 淡海 |
| K | 安坑 |

## 對象代碼

| 代碼 | 名稱 |
| --- | --- |
| TS | 列車 |

後續如果要加入倉庫、軌道、機廠設備或土建機電，可以再新增 TARGET 代碼，但不要改變既有格式。

## 流水號建議

`SEQ` 建議以 `TYPE + 工單日期 + SITE + TARGET` 為範圍各自遞增。

例如 `C-1150514-D-TS-031` 表示淡海列車在 115 年 05 月 14 日第 31 張故檢工單。若同一天建立維修工單，會另外使用 `R-1150514-D-TS-001` 起算。

這樣做的好處是各系統不會互相卡號：預檢、故檢、維修、專案可以同一天獨立產生流水號，但仍然一眼看得出來源與性質。

## 資料庫保存方式

資料庫不要只存一個字串，應同時保存拆解欄位：

| 欄位 | 用途 |
| --- | --- |
| `work_order_no` | 完整編號，例如 `C-1150514-D-TS-031` |
| `work_order_type` | `P`、`C`、`R`、`J` |
| `work_order_date` | 西元日期，用來查詢與排序 |
| `site_code` | `D` 或 `K` |
| `target_code` | 例如 `TS` |
| `daily_sequence` | 數字流水號，顯示時補三碼 |

`work_order_no` 給人看與跨系統引用；拆解欄位給資料庫查詢、排序、報表與自動產號使用。
